考虑到开发的工作量和难度，
smalljava v2.0版本不考虑实现所有的Java运算符，而是只实现其中的一部分。

计算符的优先级按照如下优先级定义
原则上：new 运算符 > 对象调用运算符 > 数学运算符 > 关系运算符 > 对象定义 > 赋值运算符【=】
在分解表达式时，最高的优先级是：
 1.赋值运算符  2.Atom表达式【()包裹】 不考虑其他运算符的存在 3.对象定义运算符
 2.关系运算符， 考虑是否存在赋值运算符
 3.数学运算符，考虑是否存在赋值运算符、关系运算符
 4.对象调用运算符，考虑是否存在赋值运算符、关系运算符、数学运算符
 5.new 运算符，考虑是否存在赋值、关系、数学、对象调用情况
 
 变量定义没有运算符
 
 运算符在计算时的优先级是反着来的，生成的AST结果已经包含了优先级的定义
